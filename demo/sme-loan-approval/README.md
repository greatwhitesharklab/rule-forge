# 小微信贷审批 Demo

## 场景说明

本演示模拟银行小微企业贷款申请的自动化审批流程。通过 RuleForge 规则引擎，实现从申请信息录入到审批结果输出的全自动化决策，涵盖信用评分、风险评估、产品匹配和额度计算等核心环节。

## 业务流程

```
申请提交 → 信息录入 → 信用评分 → 风险规则 → 产品匹配 → 额度计算 → 审批结果
```

1. **申请提交**：小微企业主提交贷款申请，提供个人及企业基本信息
2. **信息录入**：系统采集申请人年龄、收入、经营年限、信用记录、资产负债率等数据
3. **信用评分**：基于评分卡模型对申请人进行综合信用评分（满分100分）
4. **风险规则**：通过风险规则集进行自动筛查，识别高风险申请
5. **产品匹配**：根据信用评分和贷款金额匹配合适的贷款产品
6. **额度计算**：综合各项指标计算最终贷款额度和利率
7. **审批结果**：输出审批通过、人工审核或拒绝的决定

## 涉及的规则类型

| 规则类型 | 文件 | 说明 |
|---------|------|------|
| 评分卡 | `rules/credit-scorecard.json` | 基于多维度指标的综合信用评分 |
| 向导式规则集 | `rules/risk-ruleset.json` | 按优先级执行的风险筛查规则 |
| 决策表 | `rules/product-table.json` | 信用评分与贷款产品的匹配矩阵 |
| 决策流 | `rules/approval-flow.json` | 编排整体审批流程的决策流 |

## 如何运行

### 前置条件

- Docker 及 Docker Compose 已安装
- RuleForge 项目已构建（参考项目根目录说明）

### 使用 Docker Compose 启动

```bash
# 在项目根目录下启动所有服务
docker compose up -d

# 查看服务状态
docker compose ps
```

### 导入演示数据

```bash
# 等待服务完全启动后，执行导入脚本
cd demo/sme-loan-approval
chmod +x scripts/*.sh
./scripts/import-demo.sh http://localhost:8180
```

### 运行测试

```bash
# 执行全部测试用例
./scripts/test-demo.sh http://localhost:8180
```

### 访问控制台

打开浏览器访问 `http://localhost` 即可进入 RuleForge 控制台，查看和管理演示规则。

### 评分维度

信用评分卡包含 5 个评分维度：

1. **年龄**（满分25分）：25-35岁得分最高
2. **月收入**（满分35分）：收入越高得分越高
3. **经营年限**（满分30分）：经营时间越长得分越高
4. **信用记录**（满分30分）：良好记录得分最高
5. **资产负债率**（满分25分）：负债率越低得分越高

> 注意：各维度满分之和超过100分，但实际评分时会根据申请人实际情况在各维度中获得部分分数，最终综合得分上限为100分。
