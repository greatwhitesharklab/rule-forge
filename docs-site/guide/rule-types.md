# 规则类型

RuleForge 支持 7 种规则类型，覆盖业务决策的全部场景。

## 规则类型总览

| 类型 | 适用场景 | 复杂度 |
|------|---------|--------|
| 向导式规则集 | 标准 IF-THEN 条件规则 | ⭐ |
| 脚本式规则集 (UL) | 复杂逻辑，需要脚本表达 | ⭐⭐ |
| 决策表 | 多条件组合匹配 | ⭐⭐ |
| 脚本决策表 | 带脚本计算的决策表 | ⭐⭐⭐ |
| 决策树 | 层级条件判断 | ⭐⭐ |
| 评分卡 | 加权评分模型 | ⭐⭐⭐ |
| 决策流 (BPMN) | 多规则编排、流程控制 | ⭐⭐⭐⭐ |

## 向导式规则集

可视化条件-动作规则编辑器，适合简单的业务规则定义。

**适用场景**：审批规则、资格校验、业务判断

**示例**：当负债率 > 70% 时，拒绝贷款申请

## 脚本式规则集 (UL)

使用 UL 脚本语法编写规则，适合复杂逻辑表达。

**适用场景**：需要计算、字符串处理、复杂条件判断的规则

## 决策表

表格化的条件匹配，每行代表一条规则，列代表条件或动作。

**适用场景**：产品匹配、费率计算、等级划分

**金融示例**：
| 信用评分 | 申请金额 | → 产品 | 利率 |
|---------|---------|--------|------|
| ≥ 80 | ≤ 100万 | 优享贷 | 3.8% |
| ≥ 80 | > 100万 | 精英贷 | 4.2% |
| ≥ 60 | ≤ 50万 | 标准贷 | 4.8% |

## 脚本决策表

在决策表的基础上，条件和动作列支持脚本表达式。

## 决策树

树形结构的决策逻辑，从根节点出发，沿条件分支到达叶子节点。

**适用场景**：分类决策、分级审批

## 评分卡

加权评分模型，每个维度有多个分档，最终汇总得分。

**适用场景**：信用评分、风险评级、客户分群

**金融示例**：基于年龄、收入、信用记录等维度计算综合信用评分

## 决策流

基于 BPMN 2.0 标准的流程编排，将多种规则类型串联成完整决策流程。

**适用场景**：复杂业务流程、多步骤决策

**示例**：信贷审批流程 = 评分卡 → 规则集 → 决策表 → 额度计算

详见 [决策流设计器](/guide/flow-designer)。
