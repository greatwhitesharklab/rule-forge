# 评分卡

评分卡是金融风控中最常用的规则类型，通过多维度加权评分来量化风险。

## 评分卡结构

一个评分卡由多个评分维度（Attribute）组成，每个维度有多个分档（Range），每个分档对应一个分值。

### 示例：小微信用评分卡

| 维度 | 分档 | 分值 |
|------|------|------|
| 年龄 | 25-35岁 | 25分 |
| 月收入 | 8-15万 | 30分 |
| 经营年限 | 5年以上 | 30分 |
| 信用记录 | 良好 | 30分 |
| 负债率 | <20% | 25分 |
| **合计** | | **140分** |

## 创建评分卡

1. 在项目中点击「新建规则」→ 选择「评分卡」
2. 添加评分维度
3. 为每个维度设置分档和分值
4. 配置评分映射规则（如：80分以上 = "优秀"）
5. 保存并测试

## 评分映射

评分卡计算出原始分数后，可以通过评分映射表转换为等级或决策：

| 分数范围 | 等级 | 决策 |
|---------|------|------|
| ≥ 80 | 优秀 | 自动通过 |
| 60 - 79 | 良好 | 通过（需确认） |
| 40 - 59 | 一般 | 人工审核 |
| < 40 | 较差 | 拒绝 |

## 与决策流结合

评分卡通常作为决策流的一个节点使用：

评分卡计算分数 → 规则集判断风险 → 决策表匹配产品 → 输出结果
