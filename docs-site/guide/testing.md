# 规则测试

RuleForge 提供三种测试方式，确保规则在上线前经过充分验证。

## 快速测试

最简单的测试方式，输入一组数据即可查看规则执行结果。

1. 在规则编辑页面点击「快速测试」
2. 输入测试数据（JSON 格式）
3. 点击「执行」
4. 查看执行结果和规则命中路径

## 批量测试

使用预定义的测试用例集进行批量验证。

1. 准备测试用例文件（JSON 格式）
2. 在「测试」页面上传测试文件
3. 执行批量测试
4. 查看每个用例的通过/失败状态

## 仿真测试（Shadow Testing）

利用线上真实流量回放到新版本规则，对比主/陪跑差异。

- 支持 4 个维度的对比：状态、结果、输出字段、规则执行路径
- 支持 4 级严重度分类
- 不影响线上服务

详见 [架构文档 - 影子测试](/architecture/overview#影子测试)。
